# Disproid — Android タブレットを Mac の拡張ディスプレイにする

Android タブレットを、Mac の **拡張ディスプレイ（またはミラー）** として使うためのツールです。
**ワイヤレス（AirPlay）** と **有線（USB）** の 2 通りで接続できます。

| | ワイヤレス (AirPlay) | 有線 (USB) |
|---|---|---|
| Mac 側ソフト | **不要**（標準 AirPlay） | **必要**（Disproid Helper.app） |
| 接続 | 同一 Wi-Fi | USB ケーブル |
| 会場ネットの影響 | 受ける（速度・AP 分離） | **受けない** |
| USB デバッグ | 不要 | adb 方式は**必要** / **AOA 方式は不要**（実験的） |
| 表示 | ミラー / 拡張(別ディスプレイ) | **拡張（仮想ディスプレイ）**。縦持ちで縦画面に追従 |

- 映像: H.264 / H.265（自動）、最大 60fps。音声は未対応（映像のみ）。
- 検証環境: Mac (macOS 26 / Apple Silicon) ＋ Lenovo Yoga Pad Pro (Android 13)

## ダウンロード

[Releases](https://github.com/k02miu/disproid/releases) から取得できます（ビルド不要）:

- `disproid-receiver-vX.Y.Z.apk` … Android 受信アプリ
- `disproid-helper-vX.Y.Z.zip` … Mac ヘルパー（有線 USB 用、`.app`）

ソースからビルドする場合は [`android/README.md`](android/README.md) / [`mac/README.md`](mac/README.md) を参照。

## 共通: タブレットへのアプリインストール

`app-debug.apk` をタブレットに入れます（どちらのモードでも必要）。

- **USB デバッグ不要の方法**: APK をタブレットに転送してタップ → 「不明なアプリのインストールを許可」。
- **adb の方法**: `adb install -r disproid-receiver-vX.Y.Z.apk`（USB デバッグ必要）。

---

## モード A: ワイヤレス（AirPlay）

Mac 側に追加ソフトは不要。macOS 標準の AirPlay を使います。

1. **Mac とタブレットを同じ Wi-Fi（同一サブネット）に**接続。
   - 公共/社内 Wi-Fi で「AP 分離（プライバシーセパレータ）」が有効だと端末間通信が遮断され、一覧に出ません。
2. タブレットで Disproid Receiver を起動 →（解像度を選び）**「公開を開始」**。
3. Mac の **コントロールセンター → 画面ミラーリング → Disproid Receiver** を選択。
4. ミラーリングのメニューで **「別のディスプレイとして使用」** を選べば拡張ディスプレイになります。

---

## モード B: 有線（USB）

会場の Wi-Fi が遅い / AP 分離で繋がらない時向け。**Mac の仮想ディスプレイをキャプチャして USB(adb) でタブレットに送る**ため、ネットワークに一切依存しません。

### 準備（初回のみ）

1. **タブレット**: 設定 → 開発者向けオプション → **USB デバッグを ON**。
2. **Mac**: `adb`（Android platform-tools）が必要。Disproid Helper.app を配置（Releases の zip を解凍）。
3. タブレットを USB 接続 →「このコンピュータからの USB デバッグを許可」を承認。

### 使い方

1. **Disproid Helper.app を起動** → メニューバーに disproid アイコンが出る。
2. アイコンをクリック → 解像度を選んで **「開始」**。
   - 解像度は **「自動（タブレットに合わせる）」**（黒帯なし）か、各プリセット（16:9 / 16:10 / 4:3 / 3:2、縦向き 3:4 / 2:3 を含む。帯ありで任意指定）から選べます。
3. 初回は **システム設定 → プライバシーとセキュリティ → 画面収録** で **Disproid Helper** を許可 →（再度「開始」）。
4. タブレットで Disproid Receiver を開き **「USB で受信」** をタップ。
5. タブレットに Mac の仮想ディスプレイ（拡張画面）が表示されます。**タブレットを縦持ち（または縦固定）にすると縦の拡張ディスプレイ**になります。

> Mac の「システム設定 → ディスプレイ」で配置を調整できます（拡張ディスプレイとして扱われます）。

### 実験的: AOA 方式（USB デバッグ不要）

adb を介さず **AOA（Android Open Accessory）で直接 USB 通信**する方式も実装しています。
adb スタックに依存しないため接続が安定し、**USB デバッグ（開発者向けオプション）も不要**です。

- 使い方: アプリをインストールした状態で USB 接続 → ヘルパーで **「開始」** すると、ヘルパーが
  端末を accessory モードに切り替え、**Disproid Receiver が自動起動**して映像が出ます（「USB で受信」操作は不要）。
- ヘルパーは AOA 対応端末を検出すると自動で AOA を優先し、無ければ従来の adb 方式にフォールバックします。
- **現状の制限（実験的）**: 配信中の画面回転リビルドは未対応（起動時の向きで固定）、切断時の自動再接続は未対応。
  安定運用が必要な場面では adb 方式を推奨します。

## 停止

- **タブレット画面をタップ → 「停止」**（ミラー画面の操作オーバーレイ）。
- 通知シェードの停止（AirPlay モード）／アプリの「停止」。
- Mac ヘルパーのメニューバー → 「停止」／「終了」（USB モード）。

## うまくいかないとき

| 症状 | 対処 |
|---|---|
| (A) 一覧に出ない | 同一サブネットか、AP 分離が無効か、「公開を開始」済みか |
| (A/B) 映らない/黒い | いったん停止→再開。解像度を変える |
| (B) 「USB 接続失敗」 | USB デバッグ ON か、ヘルパーが「開始」済みか、`adb devices` に出るか |
| (B) ヘルパーで映らない | 画面収録の許可を確認（システム設定）。許可後に再「開始」 |
| カクつく・遅延 | 解像度を下げる（1280×720 等） |
| 音が出ない | 仕様（音声は未対応） |

## 配布物の署名について

Mac ヘルパーは現在 **ad-hoc 署名**（無料）のため、ダウンロード版は初回起動時に Gatekeeper の警告
（「開発元を確認できません」）が出ます。**右クリック → 開く** で起動できます。
警告を完全に無くすには Apple Developer Program（有料）での署名＋公証が必要です（[`mac/README.md`](mac/README.md) 参照）。
